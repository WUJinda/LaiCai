二、胜率低的根因分析

  根据之前的回测数据（宽松模式：13笔交易，胜率61.5%，但单笔最大亏损-178万），问题分三层：

  第一层：策略逻辑的内在矛盾

  你的前置条件要求：
  - 上下轨斜率同时 > 0 → 布林带整体上移 → 这是强多头趋势的信号
  - 带宽 > 21% → 波动率已经很大

  然后你在这个多头趋势中做空。

  布林带扩张+上移 = 价格在加速上涨。此时做空本质上是在逆势接飞刀。价格可能沿着上轨继续运行数根K线才回归，甚至不回归（趋势延续）。白银(ag2606)持仓41天亏损-185%就是典型案例——贵金属走的是宏观驱动的单边趋势，布林带扩张后持续沿上轨爬升。

  均值回归在趋势市中失效，而你的前置条件恰好筛选出了趋势市。

  第二层：盈亏比不对称

  - 盈利场景：价格从 > 上轨×1.02 跌回中轨 → 盈利 ≈ (上轨×1.02 - 中轨) ≈ 1个标准差宽度
  - 亏损场景：价格从 > 上轨×1.02 继续上涨，不回到中轨 → 亏损无上限

  没有止损意味着你的盈亏比是 有限收益 vs 无限风险。一笔大亏可以吃掉前面所有小赚。之前13笔交易中8笔盈利、5笔亏损，胜率61.5%看似不错，但最大亏损-178万远超任何单笔盈利。

  第三层：信号生成机制的精度问题

  - K线收盘价判断：2小时K线内价格可能剧烈波动，收盘价只是一个快照。真正的"盘中突破上轨"和"盘中跌破中轨"都被收盘价抹平了
  - 监控窗口=1根K线（Bug 1）：进一步压缩了信号生成的概率
  - 固定手数：不对品种波动率做适配，高波动品种（如银、镍）和低波动品种（如玉米）用同样的10手，风险暴露完全不同

  三、提高胜率的改进方案

  我按投入产出比排序，从低到高：

  方案A：修复已知Bug（零风险，应立即做）

  1. 修复 Bug 1：让 monitoring 持久化——前置条件满足后保持监控状态，直到趋势被打破（参考 Long 策略的状态机写法）
  2. 删除 pre_bb_upper / pre_bb_lower 死代码
  3. 修正注释（21% vs 25%）

  方案B：加入硬止损（解决盈亏比不对称）

  这是最关键的改进。建议加入两层止损：

  ┌────────────┬─────────────────────────────────┬─────────────────────────┐
  │  止损类型  │              逻辑               │        建议参数         │
  ├────────────┼─────────────────────────────────┼─────────────────────────┤
  │ 价格止损   │ 开仓后价格继续上涨超过入场价 X% │ 1%-2%（约等于1个ATR）   │
  ├────────────┼─────────────────────────────────┼─────────────────────────┤
  │ 布林带止损 │ 价格突破上轨+额外N%时止损       │ 上轨+5%（防止极端行情） │
  └────────────┴─────────────────────────────────┴─────────────────────────┘

  加入止损后，最大亏损被锁定，盈亏比从"有限vs无限"变成"有限vs有限"，胜率即便不变，期望收益也会大幅改善。

  方案C：解除入场逻辑矛盾——改为做趋势而非做回归

  你的前置条件识别的是强趋势，那就不应该做回归，而是顺势。两个方向：

  C1：顺势做多（逻辑自洽）
  - 前置条件不变（上下轨同时向上 + 带宽>21%）
  - 改为：价格突破上轨 → 做多（追趋势），价格跌破中轨 → 平多
  - 背后的逻辑：布林带扩张确认趋势启动，突破上轨是趋势加速信号，顺势跟进

  C2：做反转但加确认信号（保持做空方向）
  - 前置条件满足后，不立即入场
  - 等价格先突破上轨，然后回落再次跌破上轨（假突破确认）→ 做空
  - 背后的逻辑：突破上轨后快速回落说明多头力竭，反转概率更高

  方案D：优化出场方式

  目前的"收盘价<=中轨"出场有两个问题：滞后且只检查收盘价。

  改进：

  出场条件 = (low <= bb_middle) 或 (close <= bb_middle)

  用 low 做盘中检测：如果K线最低价已经触及中轨，即使收盘价弹回中轨之上，也触发平仓。这缩小了"实际平仓价 vs 中轨价"的偏差。

  另一种思路：分批平仓。价格到中轨平一半，到中轨-0.5×带宽再平一半。这样在急跌时可以锁定更多利润。

  方案E：增加品种过滤器

  在回测中你已经看到了——白银(ag)亏损最大，贵金属不适合这个策略。可以在策略中加入品种过滤：

  - 排除贵金属（au, ag）
  - 或加入趋势强度判断：如果过去N根K线的涨幅超过阈值（说明是强单边），跳过不做

  方案F：引入波动率自适应仓位

  替代固定10手，根据ATR动态调整：
  - 高ATR品种（波动大）→ 减仓
  - 低ATR品种（波动小）→ 可以适当加仓

  同时集成你的资金管理原则（单笔≤100万保证金）。
