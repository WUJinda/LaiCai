## 五、结论与建议

### 1. 核心结论

- 即使使用 **H2（2小时）K线数据**，原始参数（bw=0.25, bo=0.02, consec=3）在 31 个品种上**仍无法产生任何交易信号**
- 策略的"三合一前置条件"（上轨涨+下轨涨+带宽>25%）存在**逻辑矛盾**：上下轨同时上涨通常意味着带宽在缩窄，很难同时满足带宽>25%的大带宽要求
- 即使将参数放宽到极致（bw=0.10, bo=0.0, consec=1），全市场也仅产生 8 笔交易（5 个品种），信号极度稀疏

### 2. 策略逻辑的根本问题

该策略的设计思路是"捕捉布林带急速扩张后价格极端突破做空"，但前置条件的组合过于苛刻：

1. **上轨涨+下轨涨** = 布林带整体上移，但带宽带可以缩小也可以扩大
2. **带宽>25%** = 要求近期波动剧烈
3. 条件 1+2 同时满足 = 要求布林带在波动剧烈的同时整体上移，这种情况非常罕见
4. 在此基础上还要 3 根连续阳线 + 收盘价超过上轨 2%，概率趋近于零

### 3. 优化建议

**方案A：放宽参数**
- 带宽阈值降至 0.10~0.15
- 突破阈值降至 0~0.01
- 连续K线数降至 1~2
- 缺点：信号仍然很少，缺乏统计意义

**方案B：修改策略逻辑（推荐）**
- 去掉"上下轨同时上涨"的前置条件，只保留带宽条件
- 或改为"上轨上涨 + 带宽扩大"（不要求下轨也上涨）
- 增加"带宽从低位急速扩大"的触发条件（如带宽从 <15% 扩大到 >25%）
- 保留核心逻辑：连续阳线 + 突破上轨后做空

**方案C：使用更短周期K线**
- 测试 H1（1小时）或 M30（30分钟）数据
- 更短周期波动率更高，带宽更容易超过 25%
- 但也可能产生更多假信号
