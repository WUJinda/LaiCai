# 项目可视化回测机制分析

## 概述

这个项目实现了一个完整的量化交易策略回测系统，具有实时可视化K线图表功能。回测过程中可以看到策略的执行情况、K线走势、技术指标和买卖信号。

## 核心架构

### 1. 回测引擎层 (`pyStrategy/pythongo/backtesting/`)

#### 1.1 回测引擎 (`engine.py`)

**主要组件：**

- **`Engine` 类**：核心回测引擎
  - 管理账户资金、持仓、订单
  - 处理Tick数据的下载和播放
  - 实现见价成交撮合机制
  - 计算盈亏、保证金、手续费

- **`BacktestingFrame` 类**：回测框架
  - 初始化策略实例
  - 驱动Tick数据播放
  - 执行策略回调（on_tick, on_order, on_trade）
  - 生成回测结果报告

- **`run()` 函数**：回测入口函数
  ```python
  def run(
      config: Config,              # 回测配置
      strategy_cls: BaseStrategy,  # 策略类
      strategy_params: BaseParams, # 策略参数
      start_date: str,             # 开始日期
      end_date: str,               # 结束日期
      initial_capital: float       # 初始资金
  ) -> Account:
  ```

#### 1.2 数据模型 (`models.py`)

- **`Config`**：回测配置（手续费率、保证金率、数据缓存路径等）
- **`Account`**：账户模型（资金、保证金、盈亏等）
- **`Tick`**：Tick数据模型
- **`OrderDetail`**：订单明细
- **`BacktestingResult`**：回测结果（夏普比率、最大回撤、年化收益等）

#### 1.3 假对象 (`fake_class.py`)

在回测模式下，替换真实的交易接口：
- **`INFINIGO`**：模拟交易接口
- **`FakeQtGuiSupport`**：模拟Qt界面支持
- **`FakeWidget`**：模拟K线组件

### 2. 可视化UI层 (`pyStrategy/pythongo/ui/`)

#### 2.1 策略基类 (`widget.py`)

**`BaseStrategy` 类**（带UI的策略模板）：

```python
class BaseStrategy(ParentStrategy):
    """带 UI 的基础策略模版"""
    
    @property
    def main_indicator_data(self) -> dict[str, float]:
        """主图指标数据（如MA、BOLL等）"""
        
    @property
    def sub_indicator_data(self) -> dict[str, float]:
        """副图指标数据（如MACD、RSI等）"""
```

**关键特性：**
- 自动启动Qt界面线程
- 管理K线组件的生命周期
- 实时更新指标数据

#### 2.2 K线组件 (`widget.py` - `KLWidget`)

**功能：**
- 接收K线数据：`recv_kline(data)`
- 更新K线显示：`update_kline(data)`
- 加载历史数据：`load_kline_data()`
- 显示买卖信号：`update_bs_signal(price)`

**数据流：**
```
策略 → recv_kline() → 缓存数据 → load_kline_data() → 显示图表
     ↓
  on_start() 触发加载
```

#### 2.3 K线绘图器 (`drawer.py`)

**`KLineWidget` 类**：

**三个子图：**
1. **主图（K线图）**：`kline_plot_item`
   - 蜡烛图：`CandlestickItem`
   - 主图指标线（MA、BOLL等）
   - 买卖信号箭头

2. **成交量图**：`vol_plot_item`
   - 成交量柱状图

3. **底部图**：`bottom_chart`
   - 持仓量或副图指标（MACD、RSI等）

**核心方法：**
- `load_data(datas: pd.DataFrame)`：加载历史K线数据
- `update_kline(kline: KLineData)`：更新实时K线
- `insert_kline(kline: KLineData)`：插入K线（处理数据丢失）
- `set_indicator()`：设置技术指标
- `add_new_bs_signal()`：添加买卖信号箭头

**可视化技术：**
- 使用 `PyQt5` + `pyqtgraph` 实现高性能图表
- 自定义 `CandlestickItem` 绘制K线
- 十字光标 `Crosshair` 显示详细信息
- Y轴自适应缩放

### 3. 策略层 (`pyStrategy/pythongo/base.py`)

## 可视化回测工作流程

### 完整流程图

```
1. 用户编写策略
   ↓
2. 创建回测配置（Config）
   ↓
3. 调用 backtesting.run()
   ↓
4. Engine 初始化
   - 创建账户
   - 下载Tick数据
   - 订阅合约
   ↓
5. BacktestingFrame 初始化
   - 实例化策略
   - 调用 strategy.on_init()
   - 调用 strategy.on_start()
   ↓
6. 启动Qt界面（如果策略继承自ui.BaseStrategy）
   - 创建 KLWidget
   - 初始化三个子图
   ↓
7. Tick数据播放循环
   for tick in market_queue:
       ├─ 更新当前Tick
       ├─ 调用 strategy.on_tick(tick)
       │   ├─ 策略生成K线
       │   ├─ 计算指标
       │   ├─ 发出交易信号
       │   └─ 调用 widget.recv_kline()  ← 关键：发送数据到UI
       ├─ 撮合成交
       ├─ 调用 strategy.on_order()
       └─ 调用 strategy.on_trade()
   ↓
8. 实时UI更新
   - recv_kline() 缓存数据
   - on_start() 触发 load_kline_data()
   - 绘制K线、指标、信号
   - 实时刷新图表
   ↓
9. 回测结束
   - 计算最终盈亏
   - 生成统计报告
   - 保存订单明细
```

### 关键代码示例

#### 回测启动代码

```python
from pythongo.backtesting import run, Config
from pythongo.base import BaseParams
from pythongo.ui import BaseStrategy

# 1. 定义策略参数
class Params(BaseParams):
    exchange: str = "SHFE"
    instrument_id: str = "rb2505"
    # ... 其他参数

# 2. 定义策略（继承ui.BaseStrategy以获得可视化）
class MyStrategy(BaseStrategy):
    def __init__(self):
        super().__init__()
        self.params_map = Params()
    
    @property
    def main_indicator_data(self):
        return {"MA": self.ma_value}  # 返回指标数据
    
    def on_tick(self, tick):
        # 策略逻辑
        kline = self.generate_kline(tick)
        
        # 发送到UI显示
        if self.widget:
            self.widget.recv_kline({
                "kline": kline,
                "MA": self.ma_value,
                "signal_price": self.signal  # 买卖信号
            })

# 3. 配置回测
config = Config(
    access_key="your_key",
    access_secret="your_secret",
    fee_rate=0.0023,
    margin_rate=0.13
)

# 4. 运行回测
account = run(
    config=config,
    strategy_cls=MyStrategy,
    strategy_params=Params(),
    start_date="2024-01-01",
    end_date="2024-12-31",
    initial_capital=1000000
)
```

#### UI数据流

```python
# 策略中发送数据
def callback(self, kline: KLineData):
    """K线完成回调"""
    # 计算指标
    self.calculate_indicators()
    
    # 发送到UI
    self.widget.recv_kline({
        "kline": kline,              # K线数据
        "上轨": self.bb_upper,       # 主图指标
        "中轨": self.bb_middle,
        "下轨": self.bb_lower,
        "signal_price": self.signal  # 买卖信号（>0买入，<0卖出）
    })

# KLWidget接收并处理
def recv_kline(self, data):
    if self.strategy._start_flag:
        # 实时模式：立即更新
        self.update_kline_signal.emit(data)
    else:
        # 回测模式：先缓存
        self.klines.append(data["kline"].to_json())
        for indicator in self.main_indicator:
            self.all_indicator_data[indicator].append(data[indicator])

# 策略启动时加载所有缓存数据
def on_start(self):
    self.widget.load_data_signal.emit()  # 触发加载
```

## 回测与实盘的区别

### 回测模式
```python
os.environ["PYTHONGO_MODE"] = "BACKTESTING"

# 使用假对象
from pythongo.backtesting.fake_class import INFINIGO
strategy.qt_gui_support = FakeQtGuiSupport()
strategy.widget = FakeWidget()  # 或真实Widget
```

### 实盘模式
```python
# 使用真实接口
import INFINIGO
strategy.qt_gui_support = QtGuiSupport()
strategy.widget = KLWidget()
```

## 总结

这个项目的可视化回测机制通过以下方式实现：

1. **分层架构**：回测引擎、UI组件、策略层分离
2. **数据驱动**：Tick播放 → 策略计算 → UI更新
3. **Qt多线程**：独立线程运行UI，不阻塞回测
4. **信号槽机制**：使用PyQt信号实现异步UI更新
5. **高性能绘图**：pyqtgraph实现流畅的实时图表
6. **灵活扩展**：通过属性方法自定义指标显示

用户只需继承 `ui.BaseStrategy`，实现指标计算和数据发送，即可获得完整的可视化回测体验。
